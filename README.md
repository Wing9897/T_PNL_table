# PnL Matrix — 買賣損益矩陣與風險分析

以「買入價 × 賣出價」二維矩陣,視覺化每個買賣組合的損益,並結合
**信心值（K 線形態統計）** 與 **VaR / CVaR / 蝕錢機率（歷史報酬分佈）**,
輔助判斷某個價位「值不值得做 T」。

- 後端 `pnl_matrix.py`：用 `yfinance` 下載歷史股價,計算矩陣、信心值與風險指標。
- 前端 `pnl_matrix.html`：互動式矩陣（左鍵平移、滾輪縮放、十字準星、即時調整股量/佣金）。
- 桌面端 `app.py`：用 pywebview 包成原生視窗,計算結果直接在記憶體回傳前端（不需 `.json`）。

## 矩陣怎麼看

- **直軸（列）= 買入價**，**橫軸（欄）= 賣出價**，兩軸用同一組價位。
- 價位以**現價**為中心,上下展開 `price_range` 格;間距可選**固定金額 $** 或**現價百分比 %**。
- 每個**格子**顯示（對應一組買入/賣出價）：
  - 損益金額 `$`、損益率 `%`、**信心值 `%`**（達成該賣出價的機率）
- 每個**買入價（左側列標題）**顯示（對應該買入價的風險）：
  - **蝕錢機率 `%`**、**VaR 潛在虧損 `$`**、**做 T 評級**（✅/⚠️/⛔）
- 顏色：獲利綠、虧損紅、VaR 風險區琥珀、CVaR 風險區紫、現價金色。

## 名詞與計算方式（如何作準）

下面每一節先給「**是什麼**」,再給「**怎麼算**」,最後是「**可信度與限制**」。
所有公式都對應 `pnl_matrix.py` 的實際實作。

### 1. 損益（PnL）

**是什麼：** 在某買入價 `B` 買進、某賣出價 `S` 賣出 `shares` 股的淨損益（已扣佣金）。

**怎麼算：** 依佣金模式
- 百分比佣金（每邊各收）：
  ```
  買入成本 = B × 股數 × (1 + 佣金率)
  賣出收入 = S × 股數 × (1 − 佣金率)
  ```
- 固定金額佣金（每筆固定,買賣各收一次）：
  ```
  買入成本 = B × 股數 + 固定佣金
  賣出收入 = S × 股數 − 固定佣金
  ```
- 損益 `PnL = 賣出收入 − 買入成本`，損益率 `= PnL / 買入成本 × 100%`

**可信度：** 這是純會計式計算,完全準確（給定價位、股數、佣金）。前端會依你即時調整的
股數/佣金重新計算,固定佣金模式下損益率會隨股數改變（因佣金不隨股數放大）。

---

### 2. 信心值（Confidence / 達成機率）

**是什麼：** 「歷史上走勢跟現在相似時,**下一個交易日**價格落在某個賣出價附近的加權頻率」。
它衡量「**價格會不會到這個賣出價**」,屬於每一**欄（賣出價）**。

**怎麼算（K 線形態相似度加權）：**
1. 取最近 `window`（預設 5）天收盤價,減去起點 → 標準化成「形狀」（只看走勢、不看絕對價）。
2. 對歷史上每一段 5 天形狀同樣標準化,並記錄它**之後一天的價格變化量** `Δ`。
3. 用**餘弦相似度**比較「現在的形狀」與「每段歷史形狀」,得相似度 `sim ∈ [−1, 1]`。
4. 轉成非負權重 `w = (sim + 1) / 2`。
5. 把每段歷史的隔日變化投射到現價：`預測落點 = 現價 + Δ`。
6. 以權重 `w` 把所有預測落點做**加權直方圖**,落入各價位區間,再正規化成機率。

```
信心值(某賣出價) = (相似走勢的隔日落點落在此價區間的權重和) / (總權重)
```

**可信度與限制：**
- 它是**統計頻率**,不是預測保證;反映「相似形態的歷史隔日表現」。
- 只看 **5 天形狀 + 隔日**,屬**極短線**;窗口越短雜訊越大。
- 餘弦相似度只看「形狀方向」,**忽略波動幅度**;不同市況（牛/熊）一視同仁。
- 樣本越多（期間越長）分佈越穩定;事件/財報等突發無法反映。
- 與 VaR/CVaR **無關、各自獨立**——一個講「會去哪」,一個講「最壞會跌多少」。

---

### 3. VaR（風險值,Value at Risk）

**是什麼：** 在 95%（可調 `confidence`）信心水準下,**單一交易日**的「最壞跌幅」門檻。
即「正常情況下,單日跌幅有 5% 機率會超過這個數」。

**怎麼算：**
```
日報酬 r_t = 收盤_t / 收盤_{t−1} − 1        （取自所選期間,預設 1 年）
VaR%   = r 的第 (1 − confidence) 分位數      （95% → 第 5 百分位）
VaR 價 = 現價 × (1 + VaR%)
```

**可信度與限制：**
- 這是**歷史模擬法**的 **1 日 95% VaR**,直接用過去一年的日報酬分佈。
- 假設「未來分佈 ≈ 過去分佈」(平穩性);市況轉變時會失準。
- 是**單日**指標,不是「一年後的價位」;也**不代表最大可能虧損**(只是 5% 門檻)。
- 期間越短、樣本越少,分位數估計越不穩定。

---

### 4. CVaR（條件風險值 / 期望損失,Conditional VaR / Expected Shortfall）

**是什麼：** 「**一旦真的踩到最壞那 5%**」時的**平均**跌幅;比 VaR 更能描述尾部嚴重度。

**怎麼算：**
```
CVaR% = 所有 ≤ VaR% 的日報酬之平均
CVaR 價 = 現價 × (1 + CVaR%)
（若尾部樣本為空,退回 CVaR% = VaR% × 1.5）
```

**可信度與限制：**
- 比 VaR 更保守,但**更依賴尾部樣本**;極端事件本來就少,平均值噪音大。
- 同樣假設歷史代表未來;突發崩跌（黑天鵝）難以充分反映。

---

### 5. 風險值（每個買入價：蝕錢機率 + 潛在虧損 + 做 T 評級）

把上面的 VaR/CVaR 邏輯,套到**每一個買入價（列）**上,回答「**在這裡買,蝕錢風險多大、值不值得做 T**」。

**(a) 蝕錢機率 `%`**
- **是什麼：** 在買入價 `B` 買進後,**隔日仍低於 `B`（浮虧）的機率**。
- **怎麼算：** 用與 VaR/CVaR 同一份日報酬分佈,計算
  ```
  蝕錢機率(B) = P(下一日價格 < B) = (歷史日報酬 r < B/現價 − 1 的比例)
  ```
  因此 **VaR 價那一列的蝕錢機率 ≈ 5%**（與 95% VaR 對齊）。

**(b) 潛在虧損 `$`**（依目前股數即時計算）
```
VaR 潛在虧損  = B × 股數 × |VaR%|
CVaR 潛在虧損 = B × 股數 × |CVaR%|
```

**(c) 做 T 評級**（依買入價落在哪個風險區）
| 條件 | 評級 | 含義 |
|---|---|---|
| 買入價 > VaR 價 | ✅ 值博 | 正常回調區,續跌風險低 |
| CVaR 價 < 買入價 ≤ VaR 價 | ⚠️ 謹慎 | 已入 VaR 區,需明顯下跌才會到 |
| 買入價 ≤ CVaR 價 | ⛔ 不建議 | 極端尾部,通常趨勢轉壞,接刀風險大 |

**重要：兩個指標方向相反但互補**
- **蝕錢機率**：買得越**高**越大（買在高位易浮虧）;買在低位反而小。
- **做 T 評級**：買在**深跌的 VaR/CVaR 區**反而標 ⚠️/⛔。

> 例：CVaR 區某列可能顯示「蝕 1.2% 但 ⛔ 不建議」。意思是
> **價格很少跌到這麼低（短線浮虧機率低）,但一旦真的跌到,代表趨勢已壞,不適合做 T**。
> 兩者要一起看才完整。

---

### 6. 最大跌幅（Maximum Drawdown, MDD）

**是什麼：** 所選期間內,由任一高點到其後低點的**最大跌幅**,衡量歷史「最痛」的回撤。

**怎麼算：**
```
回撤_t = 收盤_t / (截至 t 的歷史最高收盤) − 1
最大跌幅 = min(回撤_t)        （取最負的一點）
```
資訊列會顯示跌幅 %,滑鼠移上可看「高點價/日期 → 低點價/日期」。

**可信度：** 為**事後**統計（已發生的歷史事實）,完全準確;但不保證未來不會更深。

## 參數說明（`pnl_matrix.py` / `app.py` 預設）

| 參數 | 預設 | 說明 |
|---|---|---|
| `ticker` | AAPL | 股票代碼 |
| `period` | 1y | 歷史資料期間（1mo/6mo/1y/2y/5y/max） |
| `confidence` | 0.95 | VaR/CVaR 信心水準 |
| `price_range` | 18 | 以現價為中心上下展開的格數 → (2N+1)×(2N+1) |
| `step_mode` | dollar | 間距模式：`dollar` 固定金額 / `percent` 現價百分比 |
| `interval` | 1.0 | 固定金額模式每格 $ |
| `interval_pct` | 0.5 | 百分比模式每格 %（現價） |
| `commission_mode` | percent | 佣金模式：`percent` 比率 / `fixed` 固定金額 |
| `commission_pct` | 0.001 | 百分比佣金（0.1%,每邊各收） |
| `commission_fixed` | 0.0 | 固定佣金（$/筆,買賣各收一次） |
| `shares` | 5 | 持股數量（前端可即時調整） |
| `window` | 5 | 信心值的 K 線形態窗口天數 |

命令列範例：
```bash
python pnl_matrix.py --ticker MSFT --period 2y --step-mode percent --interval-pct 0.5 \
    --commission-mode fixed --commission-fixed 1.0 --price-range 20
```

## 免責聲明

本工具僅作**教育與研究**用途,所有指標皆基於**歷史資料的統計推估**,
**不構成任何投資建議,也不保證未來表現**。VaR/CVaR/信心值在市況劇變、
流動性不足或突發事件下可能嚴重失準。請自行判斷並承擔風險。
